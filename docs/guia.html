1 ¿Qué hace este monitor?

El monitor analiza automáticamente los casos de repatriación para detectar si existe una denuncia policial adjunta y clasificar el tipo de estafa involucrado.

El flujo completo es:

  1. BigQuery — obtiene los REPATRI_CASE_ID de las últimas 3 semanas
  2. API de Attachments — busca si cada caso tiene archivos adjuntos
  3. Ask-to-File (IA) — analiza el contenido de cada adjunto
  4. Clasificación — detecta si es denuncia policial y el tipo de fraude
  5. Export CSV + HTML — genera los resultados
  6. GitHub — sube todo automáticamente
📅 Se ejecuta automáticamente todos los martes a las 10am, siempre que la Mac esté encendida y con VPN activa.

5 Query de BigQuery

La query obtiene los casos de repatriación de las últimas 3 semanas completas (lunes a domingo). La ventana se calcula automáticamente en cada ejecución.

SELECT DISTINCT
  a.REPATRI_CASE_ID,
  a.SITE,
  a.CREATED_AT_DATETIME,
  EXTRACT(MONTH FROM a.CREATED_AT_DATETIME) AS MES,
  a.CASE_RESULT_REASON,
  inv.CUST_ID        AS CUST_CUST_ID,
  txn.TRANSACTION_ID AS PAY_PAYMENT_ID,
  txn.AMOUNT         AS MONTO_ARS,
  txn.PRODUCT_TYPE   AS PRODUCT_TYPE,
  txn.PRODUCT_METHOD AS PRODUCT_METHOD,
  txn.ORIGIN_INSTITUTION AS BANCO_ORIGEN
FROM `meli-bi-data.WHOWNER.LK_FPR_REPATRI_CASE` a
CROSS JOIN UNNEST(INVOLVED_CUSTS)         AS inv
CROSS JOIN UNNEST(CONTESTED_TRANSACTIONS) AS txn
WHERE a.CREATED_AT_DATETIME >= DATETIME(
        DATE_SUB(DATE_TRUNC(DATE_SUB(CURRENT_DATE(), INTERVAL 1 DAY), WEEK(MONDAY)), INTERVAL 3 WEEK)
      )
  AND a.CREATED_AT_DATETIME <  DATETIME(
        DATE_ADD(DATE_TRUNC(DATE_SUB(CURRENT_DATE(), INTERVAL 1 DAY), WEEK(MONDAY)), INTERVAL 1 WEEK)
      )
  AND a.FLOW_TYPE       = 'REFUND'
  AND a.CURRENT_STATUS  = 'CLOSED'
  AND a.SITE            = 'MLA'
  AND txn.PRODUCT_TYPE   = 'bank_transfer'
  AND txn.PRODUCT_METHOD = 'cvu'

Campos clave

CampoDescripción
REPATRI_CASE_IDID del caso — se usa como externalId en la API
MONTO_ARSMonto de la transacción en ARS
BANCO_ORIGENInstitución de donde provienen los fondos
PRODUCT_METHODFiltrado por cvu — transferencias a billeteras virtuales
⚠️ La tabla usa CROSS JOIN UNNEST en dos campos anidados (INVOLVED_CUSTS y CONTESTED_TRANSACTIONS). El SELECT DISTINCT es necesario para evitar duplicados de REPATRI_CASE_ID.

6 API de Attachments

Se usan dos endpoints. La autenticación es automática por VPN corporativa — no se necesita token.

Headers obligatorios en todas las llamadas

X-Scope: fraud
X-External-System: FPR_REPATRIATION

Paso 1 — Buscar adjuntos de un caso (Search)

GET https://fpr-cross.melioffice.com/attachments/v2/files/search/{REPATRI_CASE_ID}

Devuelve un array con los adjuntos del caso. Si devuelve [], el caso no tiene adjuntos. El campo uuid de cada adjunto se usa en el siguiente paso.

Paso 2 — Analizar adjunto con IA (Ask-to-File)

POST https://fpr-cross.melioffice.com/attachments/v2/files/ask

{
  "user_prompt": "Analizá el documento...",
  "effort": "HIGH",
  "files": [{ "source": "FPR_ATTACHMENTS", "reference": "UUID_DEL_ADJUNTO" }]
}
ℹ️ Este endpoint llama a un modelo de IA — puede tardar hasta 60 segundos. El script usa timeout de 180s y reintenta hasta 3 veces con backoff exponencial en caso de error 429 o 5xx.

Access Groups requeridos

Access GroupEndpoint
attachments_prod_external_readGET search + GET download
attachments_prod_external_ask_to_file_writePOST ask

7 Clasificación de fraude

El script le pide a la IA que responda en formato JSON estructurado, lo que permite detectar tipos de fraude de forma dinámica — sin categorías fijas predefinidas.

Prompt utilizado

"Analizá el documento y respondé en formato JSON con estos campos:
- es_denuncia_policial: true/false
- tipo_fraude: string con el tipo de estafa en pocas palabras
- resumen: una oración resumiendo el motivo
- montos: lista de montos mencionados
- instituciones: lista de instituciones involucradas
Respondé SOLO con el JSON, sin texto adicional."

Categorías identificadas (abril 2026)

TipoDescripción
📞 Llamada entranteEstafador llama haciéndose pasar por banco o empresa
🔍 Búsqueda activaVíctima busca contacto en Google y llega a un estafador
📲 Acceso remotoInstalan app (AnyDesk, Ultra VNC) para tomar control del dispositivo
💳 Préstamo fraudulentoSacan préstamo a nombre de la víctima sin consentimiento
📱 Redes socialesPublicidad o perfil falso en Facebook/Instagram
👤 Cuento del tíoAlguien se hace pasar por familiar/amigo por WhatsApp
✅ Al usar clasificación dinámica con IA, el monitor puede detectar tipos de fraude nuevos automáticamente sin necesidad de actualizar el código.

9 Automatización (cron)

El script corre automáticamente todos los martes a las 10am mediante cron de Mac.

Ver la configuración actual

crontab -l

Configurar el cron (primera vez o si se pierde)

(crontab -l 2>/dev/null; echo "0 10 * * 2 /bin/bash ~/Documents/repatri-monitor/run.sh") | crontab -

Cambiar el horario

El formato es MINUTO HORA DIA_MES MES DIA_SEMANA. Días: 1=lunes, 2=martes, ... 7=domingo.

# Ejemplo: miércoles a las 8am
(crontab -l | grep -v repatri-monitor; echo "0 8 * * 3 /bin/bash ~/Documents/repatri-monitor/run.sh") | crontab -
⚠️ Requisitos para que el cron funcione: Mac encendida + VPN corporativa activa al momento de la ejecución.

10 Alertas por mail

El script envía un mail automáticamente cuando falla alguno de estos pasos:

  • VPN no activa al momento de correr
  • Error en el script Python (BQ, API o clasificación)
  • Error al hacer commit o push a GitHub

Configurar el mail destinatario

En run.sh, línea 5:

MAIL_TO="tu.email@mercadolibre.com"

11 Qué hacer si falla

Correr manualmente

bash ~/Documents/repatri-monitor/run.sh

Ver el log de la última ejecución

cat ~/Documents/repatri-monitor/output/run.log

Errores comunes y soluciones

ErrorCausaSolución
Timeout / connection refused VPN no activa Conectarse a la VPN y volver a correr
403 Forbidden en BQ Credenciales vencidas gcloud auth application-default login
403 Forbidden en API Access group no aprobado Solicitar permisos en melihelp
Authentication failed (GitHub) Token vencido Generar nuevo token en github.com/settings/tokens y actualizar con git remote set-url
pre-commit not found Hook global de Meli Usar git -c core.hooksPath=/dev/null commit

Actualizar el token de GitHub

git remote set-url origin https://TU_USUARIO:NUEVO_TOKEN@github.com/TU_USUARIO/repatri-monitor.git

Subir cambios al repo

cd ~/Documents/repatri-monitor
git add .
git -c core.hooksPath=/dev/null commit -m "descripcion del cambio"
git push origin main
